Customer Shopping Behavior Analysis
📄 Overview
This data analytics project provides an end-to-end analysis of customer shopping habits using a transactional dataset of 3,900 purchases across various product categories. The objective is to identify key spending patterns, demographic trends, and product performance metrics to guide strategic business decisions. By cleaning and preparing the data in Python , running structured relational database queries in SQL Server Management Studio (SSMS), and building an interactive dashboard in Power BI, this project delivers actionable, data-driven strategies to optimize advertising spend, loyalty programs, and subscription growth.

🚀 How to Run This Project
1. Database Setup (SQL Server Management Studio)
Open SSMS and connect to your SQL Server database engine.

In the Object Explorer, right-click on Databases and choose New Database.... Name it shopping_behavior.

Select the new database, click New Query, paste the provided SQL script containing your project queries, and click Execute (F5) to pull tables and build your views.

2. Python Environment Setup
Open your terminal or command prompt and install the dependencies required to connect to SQL Server:

Bash
pip install pandas pyodbc sqlalchemy
Open your development environment (like Jupyter Notebook or JupyterLab) and run Customer_shopping_behavior.ipynb.

This notebook automatically cleans the raw file data, processes the features, and uses pyodbc to migrate the clean dataset directly into your SSMS instance.

3. Power BI Dashboard Execution
Double-click to launch the project .pbix dashboard file in Power BI Desktop.

Click on Transform Data in the top ribbon to adjust your Data Source Settings to link with your local SQL Server connection.

Click Refresh to instantly populate all visual charts and interactive graphs with live data from your SQL database.
